Appunti Git

VARIO

$ git status (ottengo lo stato di archiviazione del repo locale/remoto)
$ git log (ottengo il log esteso di tutti i commit fatti)
$ git log --oneline (ottengo il log semplificato di tutti i commit fatti)
$ git branch -avvv (ottengo lo stato di tutti i remote e di tutti i rami)

INIZIALIZZAZIONE NUOVO REPO

$ git init (nella cartella in cui mi trovo si crea il repo)
$ git --bare init [nome cartella] (crea un repo vuoto su un server remoto, pronto a ospitare i push da git locale)

BRANCH

Creazione di un ramo
$ git branch [nome ramo] (crea un nuovo ramo)
$ git branch -v (ottiene informazioni aggiuntive sui branch)
$ git switch -c [nome ramo] (crea il ramo e si posiziona su quel ramo)
oppure $ git checkout -B [nome ramo] (crea il ramo e si posiziona su quel ramo)

Cambio di un ramo su cui sto lavorando
$ git switch [nome ramo] (cambia il ramo sul repo)
oppure $ git checkout [nome ramo]

Rinomina di un ramo
$ git branch -M [vecchio nome] [nuovo nome]

Eliminazione di un ramo
$ git branch -D [nome ramo] (cancella il ramo, per lanciare il comando devi essere fuori dal ramo)

Creazione di un ramo vuoto
$ git checkout --orphan [nome ramo] $ git rm -rf .

Ramo remoto e ramo locale
$ git branch -r (mostra il ramo remoto sul repo locale in cui siamo)
$ git branch -r (lista rami remoti)
$ git push origin -d [nome ramo remoto] (cancella ramo remoto)
$ git switch [nome ramo](collega un ramo remoto a uno locale, devi essere sul ramo che vuoi collegare)
$ git checkout --track [remote]/[nome ramo]

Procedura manuale sincronizzazione
///verifica lo stato della sincronizzazione fra repo locale e remoto
$ git remote show [nome remoto]
///collegare il ramo remoto sul repo locale, la prima volta
$ git fetch [nome remoto] [nome ramo]
$ git checkout --track [nome remoto]/[nome ramo]
///una volta i due rami sono collegati, per aggiornare il ramo sul repo locale
$ git fetch [nome remoto] [nome ramo]
$ git merge [nome remoto]/[nome ramo]
///caricamento di un nuovo branch sul repo remoto
$ git push [nome remoto] [nome ramo locale]:[nome ramo remoto]

REMOTE

Aggiungere un remote
$ git remote add [nome] [url] (di default il nome dato è origin)

Elenco di tutti i remote configurati
$ git remote -v
$ git remote rename [vecchio nome] [nuovo nome]
$ git remote remove [nome]

PUSH SU SERVER REMOTO GitHub o altro server

Quando si fa il push si carica il ramo in cui si sta lavorando su un server remoto Git.
$ git push [nome remoto] [ramo da caricare]

PULL E FETCH + MERGE

Git merge fa l’unione di due rami, unisce i rami non i commit specifici, unisce sempre dalla head corrente (unione da dove c’è la testa del ramo).
Si esegue il merge dal ramo su cui voglio unire, per esempio unire branch1 a master, vado su master e lancio git merge branch1 (unione avanti veloce); porta l’ultimo e i precedenti commit sul ramo master.
Git merge tiene sempre i rami distinti.

AHEAD AVANTI
BEHIND INDIETRO
DEATACH HEAD funziona sia in indietro (torno al commit precedente), che in avanti (vado avanti sul commit remoto).

$ git fetch (scarica il contnuto del remoto in una posizione temporanea, è più sicuro, non intacca la cartella di lavoro)
$ git fetch —all (scarico tutti i rami remoti, che rimangono remoti in attesa di merge) $ git fetch [nome remoto] (scarica tutto nel repository remoto, non nella directory di lavoro)
$ git fetch [nome remoto] [nome branch] (scarica un solo ramo)
(se tento di fare un merge con un ramo remoto e i file sono differenti, git restituisce un conflitto)
$ git pull (è l’insieme di git fetch e git merge, git pull prende un ramo alla volta per riportare le modifiche, bisogna trovarsi sul ramo da cui si vuole tirare giù le modifiche)

Ogni volta che faccio una modifica su un branch remoto, o sono a conoscenza di eventuali modifiche, meglio fare SEMPRE IL FETCH dal remoto per ottenere le ultime modifiche, poi posso decidere di unirle (merge). Ricordo che ogni modifica scaricata con FETCH è sempre tenuta in un’area protetta, non intacca la directory di lavoro.
